## Services

The `docker-compose.yml` defines four services:

| Service | Image | Purpose |
|---------|-------|---------|
| `app` | Custom PHP 8.3-FPM | Laravel application |
| `nginx` | nginx:alpine | Web server, proxies to PHP-FPM |
| `mysql` | mysql:8.0 | Database |
| `redis` | redis:alpine | Queue and cache |

---

## Development Setup

```bash
# 1. Clone and copy env
git clone https://github.com/xgenious/mini-crm.git
cd mini-crm
cp .env.example .env

# 2. Start all services
docker compose up -d

# 3. Install PHP dependencies
docker compose exec app composer install

# 4. Install JS dependencies and build
docker compose exec app npm install
docker compose exec app npm run build

# 5. Generate app key
docker compose exec app php artisan key:generate

# 6. Run migrations and seed
docker compose exec app php artisan migrate --seed

# 7. Create storage symlink
docker compose exec app php artisan storage:link
```

The app is available at `http://localhost`.

## Environment Variables

Key variables to configure in `.env`:

```env
APP_NAME="Mini CRM"
APP_URL=https://your-domain.com
APP_KEY=           # generated by key:generate

DB_HOST=mysql
DB_DATABASE=mini_crm
DB_USERNAME=mini_crm
DB_PASSWORD=secret

REDIS_HOST=redis
CACHE_DRIVER=redis
QUEUE_CONNECTION=database   # or redis

MAIL_MAILER=smtp
MAIL_HOST=your-smtp-host
MAIL_PORT=587
MAIL_USERNAME=your-username
MAIL_PASSWORD=your-password
MAIL_FROM_ADDRESS=crm@yourdomain.com
MAIL_FROM_NAME="Mini CRM"
```

## Queue Worker

Email notifications and other background jobs run via Laravel queues. Start the worker:

```bash
docker compose exec app php artisan queue:work
```

In production, use Supervisor to keep it running. A `supervisor.conf` is included in the repo root:

```bash
# Copy to Supervisor's conf.d directory
sudo cp supervisor.conf /etc/supervisor/conf.d/mini-crm.conf
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start mini-crm:*
```

The Supervisor config manages two processes:

| Process | Command |
|---------|---------|
| `queue-worker` | `php artisan queue:work --sleep=3 --tries=3` |
| `scheduler` | `php artisan schedule:work` |

## Task Due Reminder Scheduler

The `SendTaskDueReminders` command runs daily at **08:00 UTC** via the Laravel scheduler. With Supervisor running `schedule:work`, this happens automatically.

To run it manually:

```bash
docker compose exec app php artisan app:send-task-due-reminders
```

## Manual Server Deployment

If Docker is not available, deploy to a bare server with PHP 8.2+:

```bash
# 1. Install dependencies
composer install --no-dev --optimize-autoloader
npm install && npm run build

# 2. Configure environment
cp .env.example .env
php artisan key:generate
# Edit .env with your database and mail settings

# 3. Run migrations
php artisan migrate --seed

# 4. Set permissions
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

# 5. Create storage symlink
php artisan storage:link

# 6. Cache config and routes
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### Nginx config

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/mini-crm/public;

    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
    }
}
```

### Add Laravel scheduler to crontab

```bash
* * * * * cd /var/www/mini-crm && php artisan schedule:run >> /dev/null 2>&1
```
